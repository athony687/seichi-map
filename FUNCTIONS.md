# アプリ機能一覧（FUNCTIONS.md）

ユーザーが実際に触れる機能をカテゴリ別にまとめたドキュメント。
実装の詳細（コンポーネント名・state 名など）は STRUCTURE.md を参照。

最終更新: 2026-07-04

---

## 1. 位置情報・コンパス許可

| 機能 | 詳細 |
|---|---|
| 起動時の許可カード | LIVE モードで起動するたびに「Location & Compass」ボトムシートを表示 |
| Allow ボタン | `getCurrentPosition`（位置情報）と `DeviceOrientationEvent.requestPermission`（コンパス）を同時にトリガー。ボタン押下がユーザージェスチャーとして扱われるため iOS も正常動作する |
| Use without location | 許可をスキップ。GPS・コンパスなしでアプリを利用可能 |
| GPS 同意の永続化 | Allow したかどうかを localStorage に保存（`seichi_location_consented`）。次回起動時はカードが出た直後、GPS が自動起動する |

## 2. 起動時ダッシュボード

| 機能 | 詳細 |

## 3. 地図

| 機能 | 詳細 |
|---|---|
| ライト / ダークテーマ | 設定画面からライトマップ（暖色系）とダークマップ（ネイビー・紫系）を切替。選択は localStorage に保存（`seichi_map_theme`） |
| 移動制限 | 日本国内（北緯 23〜46.5°、東経 121〜155°）から地図が出ないよう制限 |
| 初期表示 | 起動時は東京付近（zoom 6）を表示 |
| ジェスチャー操作 | スワイプ・ピンチ・ダブルタップで自由にパン・ズーム |
| ズームボタン非表示 | `disableDefaultUI` でズーム UI を非表示にし、地図面積を最大化 |
| Google ロゴ常時表示 | コントロールバーを Google ロゴの上にかぶせないよう配置（利用規約準拠） |
| ヘッダー | 画面上部に黒背景で「Animap.jp」ロゴを固定表示（高さ 50px、z-index 1000） |

## 6. 近接フローティングラベル

| 機能 | 詳細 |
|---|---|
| ProximityLabel | 現在地から 500m 以内の聖地ピン真上に、地図に重なるフローティングラベルを表示 |
| ラベル内容 | アニメ名（小文字大文字混在、薄いキャップ表示）＋ポップな一言フック（6パターンからハッシュで固定割当） |
| ポップフック例 | "You're literally inside [Anime] right now 🌸" など |
| 複数スポット対応 | 複数の近接スポットがあっても、各ピン上に個別ラベルを同時表示できる |
| タップでカード展開 | ラベルをタップすると、そのスポットの完全カードが直接展開される（一度閉じてから開くようなことはない） |
| カード表示中の扱い | カード表示中のスポットはラベルを非表示にし、カードと二重にならない。カードを折りたたんで戻った場合は再表示 |

## 7. 聖地カード（二段階表示）

| 機能 | 詳細 |
|---|---|
| 完全カード表示 | ラベルタップ・ピンタップ・検索・ダッシュボード経由で聖地カードを全開で表示 |
| 折りたたみ | ヘッダー部（アニメ名・スポット名・エリア・距離）は常時表示。タップで本文を展開 |
| 現在地からの距離 | GPS または仮想マーカーからの直線距離を表示（例: About 1.2km） |
| 天気適性メッセージ | 現在の天気・時間帯に応じた訪問アドバイスを表示（sunny / cloudy / rainy / evening） |
| AI 生成紹介文 | Claude Haiku が生成した英語紹介文を展開エリアに表示 |
| ローディング表示 | 紹介文生成中は小さなスピナーと "Generating introduction…" を表示 |
| AI / 静的バッジ | 生成成功時は「✨ AI generated」、失敗時は「📄 description」を表示 |
| お気に入りボタン | カード内のハートボタンでお気に入り登録・解除。localStorage に永続化（`seichi_favorites`） |
| 閉じるボタン | カード右上の ✕ ボタンで閉じる（`selected` と `cardExpanded` を両方リセット） |

## 10. AI 紹介文生成

| 機能 | 詳細 |
|---|---|
| リアルタイム生成 | カードを開くたびにバックエンド（FastAPI + Claude Haiku）へリクエストを送信 |
| パーソナライズ | ユーザーの好み（nickname / familiarity / mood / travelStyle）に合わせてプロンプトを調整（詳細は「15. パーソナライズ」参照） |
| フォールバック | 生成失敗時は `generic_intro_en`（カード別汎用文）または `intro_short_en` を表示。どちらも未設定なら汎用英語テキストを表示 |
| キャッシュ | 生成済みの紹介文はクライアント変数・サーバーメモリの両方にキャッシュされ、同じスポットの2回目以降は API を呼ばない（prefs なし時のみ） |

## 11. LIVE モード（デフォルト）

| 機能 | 詳細 |
|---|---|
| GPS 追跡 | `watchPosition` でリアルタイムに現在地を取得し、青い丸マーカーで地図に表示 |
| コンパス方向表示 | スマホの向きをリアルタイム取得し、現在地マーカーに約60°の扇形コーンで表示。RAF ループで補間スムージングあり |
| 近接自動ラベル表示 | 聖地から 500m 以内に入ると、ピン真上にフローティングラベルを自動表示 |
| 近接自動カード表示 | 聖地ピンをタップ or ラベルをタップするとカードが展開 |
| カメラ追従 | カード表示時はそのスポットへカメラがズーム。カードを閉じてもカメラはその場に留まる |
| GPS ボタン | 画面右下に表示。GPS 取得状況を色で表示し、タップで現在地へカメラを移動 |
| 位置取得中のローディング | 同意後に GPS 取得が完了するまで（最大 8 秒）全画面の白いローディング画面を表示 |
| DEMO への切り替え | 左下コントロールバーの「DEMO」ボタンで DEMO モードに切り替え |

## 12. DEMO モード

| 機能 | 詳細 |
|---|---|
| LIVE への切り替え | 左下コントロールバーの「LIVE」ボタンで LIVE モードに切り替え |
| 開始地点設定 | 「📍 Set Start」ボタン押下後、地図をタップして仮想マーカーの出発点を設定 |
| 変更ヒント | デモモード起動直後、Set Start を促す吹き出しが表示される |
| 自動ルート生成 | 開始地点から半径 40km 以内の聖地を、近い順に自動でルート設定 |
| 再生 / 一時停止 | ▶️ / ⏸ ボタンで仮想マーカーの移動を制御（開始地点未設定時はボタンが無効） |
| 仮想マーカー移動 | 紫の丸マーカーが 600ms ごとに約 110m ずつ自動移動 |
| 近接ラベル表示 | 仮想マーカーが聖地から 500m 以内に入るとフローティングラベルを表示 |
| 自動カードクローズ | 仮想マーカーが聖地から 500m 以上離れるとカードを自動クローズ |
| スポット到達後の停止 | 各スポット到達後は 5 秒停止してから次のスポットへ移動 |
| カメラ追従 | 仮想マーカーを常にカメラが追跡 |

## 13. 天気情報

| 機能 | 詳細 |
|---|---|
| 自動天気取得 | Open-Meteo API（無料・APIキー不要）で現在地付近の天気を自動取得。許可不要 |
| グリッドキャッシュ | 0.5° グリッド単位でキャッシュし、20 分間は再取得しない |

## 14. 検索機能

| 機能 | 詳細 |
|---|---|
| 検索バー | ヘッダー右側（"Search here"）にアニメ名を入力 |
| 候補表示 | 入力に部分一致するアニメ名を最大 6 件ドロップダウン表示 |
| ピン強調 | アニメを選択するとそのアニメの聖地のみ赤ピンで表示。他の聖地はグレーに減光 |
| バウンスアニメーション | 検索直後、ヒットしたピンが 2.5 秒間バウンスして場所を知らせる |
| リセット | 検索バー右の ✕ ボタンで検索クリア → 元の全ピン表示に戻る |
| キーボード操作 | Esc キーで候補を閉じる |

## 15. 初回アンケート

| 機能 | 詳細 |
|---|---|
| 表示条件 | 初回起動時（localStorage に好みデータがない場合）のみ全画面表示 |
| ステップ 1 | ニックネーム入力（任意。後でカードの呼びかけに使用） |
| ステップ 2 | アニメへの馴染み度（Newcomer / Casual fan / Big fan） |
| ステップ 3 | 今の気分（Emotional / Exciting / Heartwarming / Romance） |
| ステップ 4 | 旅のスタイル（Taking photos / Relaxed walking / Visiting many spots） |
| 保存 | 入力完了後に localStorage へ保存。次回以降はアンケートを省略 |

## 16. 設定画面

| 機能 | 詳細 |
|---|---|
| 開き方 | 左下コントロールバー左端の ⚙️ ボタン |
| アコーディオン形式 | 各項目はタップで開閉。一度に1項目のみ展開される |
| ユーザー設定 | nickname / familiarity / mood / travelStyle（アンケートと同じ 4 項目） |
| 天気手動設定 | sunny / cloudy / rainy / evening から選択。「Auto」で自動取得に戻る |
| マップテーマ | ライト（暖色系）/ ダーク（ネイビー・紫系）を切替 |
| このアプリについて | アプリ概要・データ出典・注意事項を表示するセクション |
| 保存 | 「Save」ボタンで localStorage を更新。カードキャッシュを即時クリア |
| リセット | 「Reset」ボタンで全設定を削除し、初回アンケートに戻る（確認ステップあり） |
| キャンセル | ✕ ボタンで変更なく閉じる |

## 17. パーソナライズ紹介文

ユーザーが設定した好みに応じて、Claude Haiku へのプロンプトが変わる。

| 設定 | 値 | 紹介文への影響 |
|---|---|---|
| `nickname` | 任意の文字列 | 紹介文の冒頭で名前を呼びかける（例: "Hey Taro,"） |
| `familiarity` | Newcomer | アニメ未知の読者向けに、前提知識なしで説明 |
| `familiarity` | Casual fan | 親しみやすく自然にシリーズを紹介 |
| `familiarity` | Big fan | 特定シーンやキャラクターに踏み込んだ深い解説 |
| `mood` | Emotional | 感動的・心を揺さぶる側面を強調 |
| `mood` | Exciting | 躍動感・興奮を強調 |
| `mood` | Heartwarming | 温かく穏やかな雰囲気を強調 |
| `mood` | Romance | ロマンチックな景観・シーンを強調 |
| `travelStyle` | Taking photos | 写真映えするポイントを提案 |
| `travelStyle` | Relaxed walking | ゆったり楽しむ方法を提案 |
| `travelStyle` | Visiting many spots | 立ち寄る価値のあるポイントを簡潔に紹介 |

> ⚠️ 好みが設定されている場合、クライアント・サーバー両方のキャッシュをバイパスして毎回生成する。
